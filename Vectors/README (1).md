# LangChain Vector Stores (Chroma)

Hands-on exploration of **Vector Stores** in LangChain — storing document embeddings and performing similarity search using Chroma.

Part of my [LangChain Fundamentals](../) learning series.

## 📌 Overview

Vector stores hold document embeddings and let you search for semantically similar content. This notebook walks through setting up a local Chroma vector store, adding documents, and querying it.

## 🛠️ Tech Stack
- Python 3.x
- LangChain / langchain-community
- ChromaDB (local persistent vector store)
- langchain-huggingface (embeddings)
- Jupyter Notebook

## 📂 Folder Structure
```
langchain_vector/
├── langchain_chroma.ipynb
├── requirements.txt
├── .env              # not pushed (see .gitignore)
├── .gitignore
└── README.md
```
*(local-only, not pushed: `my_chroma_db/` — the generated vector database files, regenerated by running the notebook)*

## ⚙️ Setup & Installation

```bash
git clone https://github.com/somil02/Langchain.git
cd Langchain/langchain_vector
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## 🔑 Environment Variables

Create a `.env` file in this folder:
```
HF_TOKEN=your_key_here
OPENAI_API_KEY=your_key_here
```
The notebook loads these via `python-dotenv` — never hardcode tokens directly in notebook cells.

## ▶️ Usage

```bash
jupyter notebook langchain_chroma.ipynb
```
Running the notebook will regenerate the `my_chroma_db/` folder locally.

## 📖 What I Learned
- How Chroma persists embeddings locally to disk for reuse across sessions
- Adding, querying, and updating documents in a vector store
- Why API keys should always be loaded via `.env` + `python-dotenv`, never hardcoded in notebook cells — hardcoded tokens get committed permanently into notebook history since notebooks are plain JSON files
